Creating my own website

This page walks through how I set up my own website to document my projects

To create my own website, I generally followed Jake Wright's youtube guides on HTML and CSS and only edited some parts to suit my own preferances. Thus, my website will look very similar to his example.

Jake Wright's youtube guides: youtube channel

  • Learn HTML in 12 minutes
  • Learn CSS in 12 minutes

Files & folders

I have a main folder that contains everything related to this project. Inside the main folder, I have the files for my three main pages, the index, about and projects page, and the CSS files that is used to style the website. There are also sub-folders that contains all other files, such as a images folder and a folder to contain all my dedicated project documentaion pages. For every different content in my website, there will be a new file and page created, and pages with similar content are categorised to keep things organised and easy to understand

Below will explain the layout of my website pages and some of the HTML functions used to create it.

Writing in HTML (Hypertext Markup Language)

Basic layout

Everything in my website and the layout choices are kept simple, straighforward and organised neatly to provide ease of use and navigation.
I have a main website header and a navigation bar present at the top across all pages so that the layout will be consistent.
Only the main contents of the page below that will change, and all pages have a header at the start to introduce the content of the page.

Home/Index page

The home page is the only page with a different layout to keep it from being too boring. Since the home page also does not have much content on my website, I decided that it would look better with everything centralised. Since I wanted to centralise everything in the home page, I wrote as if it were and would later shift and organise with CSS.
I used a table to list out items in my navigation bar horizontally and later used CSS to space them out.

A simple page header, "HOME", was used and bars were added to both sides to fill up the page. Then a simple and straightforward introduction to the use of the website below the header is put right below.

I also decided to display my EP1000 certifications on the home page, such as the passing of the lab safety and 3D printing online test, so that it can be easily seen for the module assessment. This may be later changed after completion of the module.

About page

In my about page, I put a profile picture and used a table function to wrap text containing some basic personal details to the side of it, similar to an IC card or ID part of a passport. Below that I put a thin line across the page to seperate the rest of the about page that will provide further introduction to myself.

Below the line, I gave a short write-up about myself and used an unordered list below that to list some of my interests.

Projects page

I decided that my main projects page will just list out most projects that I have done or will be doing, as well as provide links to the dedicated documentation page. So I put a page header and gave a very short introduction to the page with a paragraph function at the top. I then used headers to sort and categorise based on methods used or type of project, and used an unordered list function to list out the projects I have done. Each list item is a link to the project documentaion page, done by using the <a> function in the list item.

Styling with CSS (Cascading Style Sheets)

After creating a working webpage in HTML. A CSS text file was then used to make my web pages look better and not just plain text against a white background. CSS allowed me to set a border for different text bodies, shift text bodies to different parts of the pages, change background colours. I could also further edit the look of the page content, such as text alignment, font type, font colour and font size.

Since all pages in my website follows the same layout except for the home page, I created two CSS text files: one for the home page, and a second one for everything else.

Editing the HTML files

Before writing the CSS files, the first step I had to do was to put everything in my HTML files into divisions using the <div> function. This allows each part to be edited, styled or shifted independently without unintentionally affecting other parts.

Following Jake Wright's CSS guide, I have:

  • <div id="container"> which contains all parts inside the <body> commands of the HTML file and all other divisions.
  • <div id="header"> which contains the website header
  • <div id="content"> which contains the nav and main divisions
  • <div id="nav"> which contains the navigation bar and the functions used for it
  • <div id="main"> which contains the main contents of the page
  • <div id="footer"> which contains the website footer, which contains a copyright claim following Jake Wright's example
